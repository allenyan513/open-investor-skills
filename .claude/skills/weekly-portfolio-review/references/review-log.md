# 历史复盘记录格式

> 记录每次复盘的判断，定期回头验证准确性。
> 这是提升投资判断力最重要的习惯之一。

---

## 为什么要记录复盘日志？

```
不记录 → 只记住自己"猜对"的判断，忘记"猜错"的
记录   → 强制看到真实的判断准确率 → 发现系统性偏差 → 改进
```

---

## 复盘记录格式

每次复盘后，在 `output/review-log.md` 追加一条记录：

```markdown
## [日期] 第 [X] 周复盘记录

### 本次复盘判断

| 股票 | 判断 | 核心依据 | 验证节点 |
|------|------|---------|---------|
| NVDA | 继续持有 | AI 算力需求持续，GTC 大会催化 | 2026-05-28 Q1 财报 |
| ADBE | 减仓观察 | CEO 离职 + 指引弱，逻辑部分弱化 | 新 CEO 公布时间 |
| MSFT | 继续持有 | Azure 增速稳定，Copilot 商业化加速 | 2026-04-30 Q3 财报 |

### 本周操作记录（如有）
- [操作描述，如"ADBE 减仓 50%，均价 $248"]
- [操作理由]

### 本周市场背景简记
[1-2 句话记录影响持仓的宏观事件]

---
```

## 判断验证记录格式

**每隔 4-6 周**，回头看之前的判断是否正确：

```markdown
## [日期] 验证记录 — 回顾 [X] 周前的判断

| 股票 | 当时判断 | 实际发生 | 判断准确？ | 复盘反思 |
|------|---------|---------|-----------|---------|
| NVDA | 继续持有 | 财报超预期，+18% | ✅ 准确 | 逻辑验证有效 |
| ADBE | 减仓观察 | 新CEO背景强，股价反弹 | ⚠️ 部分准确 | 减仓时机略早，但风控合理 |
| MSFT | 继续持有 | Azure 增速放缓，-8% | ❌ 失误 | 低估了市场对增速边际变化的敏感度 |

### 本轮判断准确率
准确：[X] 只 / 总计 [X] 只 = [X%]

### 系统性反思
[我的判断中是否有某类系统性偏差？例如：
 - 总是高估成长逻辑的持久性？
 - 总是忽略宏观利率的影响？
 - 在盈利仓位上太保守，亏损仓位上太宽容？]
```

## 长期统计模板

每季度整理一次长期统计：

```
季度复盘统计（[年份] Q[X]）

判断准确率：
  继续持有 → 实际上涨或持平：___/%
  减仓观察 → 实际有效规避下跌：___/%
  清仓建议 → 实际规避了亏损：___/%

最成功的判断：[描述]
最大的失误：[描述]
需要改进的判断模式：[描述]
```

---

## 使用说明

1. 每次运行 `weekly-portfolio-review` Skill 后，将汇总结果追加到 `output/review-log.md`
2. 命名保持固定文件名（不像其他报告按日期新建），便于长期积累
3. 每 4-6 周做一次「验证记录」，对照当时判断看实际结果
4. 每季度做一次「长期统计」，发现自己的判断偏差模式
